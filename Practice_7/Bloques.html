Diagrama de Arquitectura de Microservicios

1. Patrón: API Gateway

El API Gateway actúa como un punto de entrada único para todas las solicitudes del cliente. Simplifica la comunicación, maneja la autenticación y enruta las solicitudes al microservicio correcto. El cliente no necesita saber dónde vive cada servicio.

2. Patrón: Database per Service

Cada microservicio (Pedidos, Clientes, Productos) es dueño exclusivo de su propia base de datos. Ningún otro servicio puede acceder directamente a la base de datos de otro. La comunicación *debe* hacerse a través de la API del servicio. Esto garantiza un bajo acoplamiento.
